# About Page Mockups Documentation

**Total Mockups Identified: 7**

This document provides a detailed analysis of the visual mockups and component structures used in the `About` page.

## 1. Hero Section Tablet Display
**Title:** 3D Tablet Visualization
**Description:** A 3D-transformed tablet device frame situated in the Hero section. It features a realistic bezel, "Dynamic Island" top bar, and a screen area that displays an application screenshot (`/tablet.png`). The device rotates slightly on hover.
**Dimensions:**
- **Device Frame:** 360px (Width) x 520px (Height)
- **Screen Content Area:** Approximately 352px x 512px (Inset by border borders).
- **Aspect Ratio:** ~0.69:1
**Usage Tip:** Use this mockup to display high-fidelity screenshots of the mobile application or dashboard interface. Ensure the source image handles the rounded corners (2.5rem radius) gracefully.

## 2. Liverpool Origin Map
**Title:** Genesis Morphing Graphic
**Description:** A large visual container featuring an abstract map background (SVG paths) and a pulsing dot animation representing a specific location (Liverpool). Includes a glassmorphism quote overlay at the bottom.
**Dimensions:**
- **Desktop (lg):** Height fixed at 600px, width scales with column.
- **Mobile/Tablet:** Aspect function `aspect-square` (1:1 ratio).
- **Corner Radius:** `rounded-[3rem]` (48px).
**Usage Tip:** Use this to visualize geographic origins, headquarters, or server locations. The pulsing animation draws attention to a specific focal point on a map.

## 3. Evolution Timeline Cards
**Title:** Timeline Phase Card
**Description:** Interactive cards positioned along a vertical timeline. They feature an icon badge, title, and rich text description. The cards have a glass-hover effect and glow based on the phase color.
**Dimensions:**
- **Container:** High flexibility, generally half the width of the `max-w-5xl` container minus padding.
- **Padding:** `p-8` (32px) to `p-10` (40px).
- **Corner Radius:** `rounded-3xl` (24px).
**Usage Tip:** Use for storytelling, step-by-step process flows, or chronological history. Best suited for content with a clear icon + title + description structure.
